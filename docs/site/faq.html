Is TorrentOS free? +
Yes. TorrentOS is fully open source under the GPL-3.0 licence. You can use it, modify it, and share it for free. The source code is on GitHub.
What are the system requirements? +
Minimum: 8 GB RAM (16 GB recommended), a 4-core CPU, 50 GB storage, a 64-bit x86 processor, and a GPU with Vulkan support. Most hardware made in the last 5–6 years meets these requirements out of the box.
Can I install TorrentOS alongside Windows? +
Yes — dual boot works. Run the TorrentOS installer, select your free disk space as the install target, and keep your existing EFI partition. The bootloader will give you a choice between TorrentOS and Windows on every startup. A full step-by-step guide is available on the wiki.
Does it work on a laptop? +
Yes. WiFi (via NetworkManager), touchpad gestures (libinput), screen brightness, and suspend/resume all work out of the box on most laptops. NVIDIA discrete GPUs require one extra step after installation — see the NVIDIA question below.
How do I install NVIDIA drivers? +
TorrentOS ships with full NVIDIA support pre-configured — nvidia-dkms, the Wayland GBM backend, DRM modeset, and hardware cursor workaround are all set up automatically. On the live ISO just run torrentos-doctor to verify everything is working. If you installed manually and the driver is missing, run sudo pacman -S nvidia-dkms nvidia-utils and reboot.
Does TorrentOS support Apple Silicon (M1, M2, M3, M4, M5)? +
Yes! TorrentOS supports Apple Silicon via the Asahi Linux project. The ISO is x86_64 only, but you can run TorrentOS on M1/M2/M3/M4 Macs by first installing Arch Linux (Asahi), then running our one-command bootstrap script. M4 Pro/Max have early support; M5 is coming soon. The whole process takes about 20 minutes. Read the full Apple Silicon guide →
Will my MacBook's trackpad gestures work? +
Yes. On Apple Silicon Macs, multi-touch gestures are configured via libinput-gestures — three-finger swipe changes workspaces, pinch-to-zoom works in compatible apps. The config is at ~/.config/libinput-gestures.conf. On PC laptops, gestures also work through the same libinput backend.
How do I update TorrentOS? +
Click the update icon in the taskbar to open torrentos-update-gui — it shows available updates and applies them with one click. Or run sudo pacman -Syu in any terminal. TorrentOS is a rolling release, so updates are continuous rather than version-based.
Can I use Flatpak apps? +
Yes. Flatpak is pre-installed and Flathub is configured as the default remote. Install apps via GNOME Software, the Discover app, or from the terminal: flatpak install flathub <app-id>. Popular creative apps like DaVinci Resolve, Kdenlive, and Inkscape are all available this way.
What desktop environment does it use? +
TorrentOS uses Hyprland — a dynamic tiling Wayland compositor — with a macOS-style layout. The taskbar is Waybar, the app launcher is rofi, and notifications are handled by swaync. The whole stack is Wayland-native and GPU-accelerated.
Is gaming supported? +
Yes. GameMode and MangoHud are pre-installed for performance overlays. Install Steam from Flatpak (flatpak install flathub com.valvesoftware.Steam) or pacman (sudo pacman -S steam), then enable Proton in Steam Settings → Steam Play to run Windows games. Vulkan is set up by default. NVIDIA RTX/GTX cards work out of the box with the bundled drivers.
How do I report a bug? +
Open an issue on github.com/rupertjwakefield2-dot/torrentos/issues. Before you do, run torrentos-doctor from the terminal — it collects system info and outputs a report you can paste directly into the issue.
What's the difference between v0.4 and v0.5? +
v0.5 adds the brand-new TorrentOS Settings app (libadwaita, covers appearance, display, accessibility, and dev mode), a Developer Mode installer that sets up Rust, Node, Docker, and VS Code in one click, significantly improved accessibility options, and a polished first-boot setup wizard.
Can I go back to Arch Linux defaults? +
TorrentOS is Arch Linux — it's built directly on the Arch base without any forks or incompatible modifications. Every package from the official Arch repos and the AUR works normally. You can install or remove any package freely, and the system stays fully Arch-compatible.
